## What this is

A Kubernetes operator that manages deployments of [New Relic's nri-bundle Helm chart](https://github.com/newrelic/helm-charts/tree/master/charts/nri-bundle). It exposes a `Monitor` CRD (`newrelic.com/v1alpha1`) and an `NRIBundle` CRD. When a `Monitor` resource is applied, the operator downloads the specified version of `nri-bundle` from `https://helm-charts.newrelic.com` and runs a Helm reconciler for it.

Built on [Operator Framework's Hybrid Helm Operator](https://github.com/operator-framework/helm-operator-plugins) and `sigs.k8s.io/controller-runtime`.

## Commands

```bash
# Build the binary
make build

# Run tests (downloads envtest assets, runs go test ./...)
make test

# Run a single test package
KUBEBUILDER_ASSETS="$(./bin/setup-envtest use 1.28.3 -p path)" go test ./controllers/... -v -run TestAPIs

# Format and vet
make fmt vet

# Regenerate CRD manifests and DeepCopy methods after editing api/v1alpha1/newrelic_types.go
make manifests
make generate

# Regenerate Helm chart templates (set IMG to your image)
make helm IMG=<registry>/newrelic-operator:<version>

# Build Docker image (single-arch, runs tests first)
make docker-build IMG=<registry>/newrelic-operator:<version>

# Build multi-arch Docker image (no test run)
make docker-multiarch IMG=<registry>/newrelic-operator:<version>

# Deploy to cluster in current kubeconfig context
make deploy IMG=<registry>/newrelic-operator:<version>

# Tear down
make undeploy
```

## Architecture

### Reconciliation model

The operator uses a two-level manager pattern:

1. **Outer manager** (`main.go`): Runs `NewRelicReconciler`, which watches `Monitor` CRs.
2. **Inner HelmManager** (`controllers/newrelic_controller.go`): A second `ctrl.Manager` spun up dynamically inside `NewRelicReconciler` when a `Monitor` CR is created or its `.spec.version` changes. This inner manager runs the Helm operator reconciler for `NRIBundle` CRs.

When `.spec.version` changes, the inner manager is cancelled and replaced with a new one pointing at the new chart version. When the `Monitor` CR is deleted, the inner manager is stopped.

`chart_utils.go` handles downloading the `nri-bundle` Helm chart from the New Relic Helm repository into a temp directory, using the Helm Go SDK directly.

### Tests

Tests use `envtest` (real API server, no kubelet). The test suite bootstraps with CRD paths from `config/crd/bases/`. Run `make manifests` before `make test` if CRDs are stale.

### Deployment namespace

The operator runs in `newrelic-k8s-operator-system` by default (hardcoded in `make deploy` and kustomize config).
